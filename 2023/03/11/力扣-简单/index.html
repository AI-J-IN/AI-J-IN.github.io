两数之和

暴力解法

通过遍历依次 将 nums下标为 i 的数据拿出来，然后将tager-下标为 i 的数据的差值，用来判断 该数值是否 存在于 nums 下标为 i+1到 最后的 范围里，如果存在就返回 对应的 下标 i 和 nums[i+1:].index(a)+i+1

为什么这里是 nums[i+1:].index(a)+i+1 ,因为后面的下标是在 nums[i+1：] 这里拿出来的，在实际的nums中 要将 i+1 再加回去，

肯定有人要问为啥 不直接 nums.index(a) 这样不是也可以去到数据对应的下标吗，为何多此一举

比如 nums=[3,3] ，target=6, 如果使用 nums.index(a) 那 return 的结果就是 [0.0]，而不是[0.1]，因为index 通过数值 去取对应的下标 默认都是取 第一个匹配到的值，后面再有也不会匹配到了，

所以 这个问题的根本原因就是 如何 通过index 去取下标时，能避开已经 得到的下标，从而去取后续的下标，

一种方法就是 nums[i+1:].index(a)+i+1 通过 切分， 让 nums[i+1:].index(a)+i+1在 nums[i+1:] 这里面去查找，这样就可以避开 下标为 i 的情况，因为后续通过 index 的数据 肯定不包含 i 这个下标了，因为是在 nums[i+1:]里去 查询的 所以对比 nums 需要将再加上 i+1 让 nums[i+1:].index(a)+i+1得到的下标 等于 nums 的下标

第二种方法是 根据上面所述 如何 通过index 去取下标时，能避开已经 得到的下标，从而去取后续的下标，我们可以将得到的下标通过 nums[i]=3333（3333是随便一个数值，是要不影响我们得到后续的下标就行），现在nums.index(a) 就可以直接 避开 i，去取后面值的下标了

排序+双指针

​ 因为 nums里面的数据是无序的，所以我们需要使用copy()，复制一个tmp出来，并使用 sort 进行排序。

​ 这里要创建两个变量 i 和 j ，从tmp的两头往中间靠拢，也就是所谓的 双指针，所以可以写个 while i<j

因为i 一定是要比 j 小的，否则取的数就会重复，

​ tmp里的数据都是从小到大排序的,所以

​ (tmp[i]+tmp[j])>target 时说明 加起来的 数值太大了，需要减少，所以 需要移动 j 往中间靠拢一个位置，即 j=j-1。

​ (tmp[i]+tmp[j])<target 时说明 加起来的 数值太小了，需要增加，所以 需要移动 i 往中间靠拢一个位置，即i=i+1,

​ .如此循环，直到 (tmp[i]+tmp[j])==target，即 结束循环，表示已经拿到了想要的数据的

​ p=nums.index(tmp[i]) 即获得 对于 tmp[i]的下标

​ k=nums.index(tmp[j]) 即获得 对于 tmp[j的下标

​ 为啥 中间会有个 nums.pop(p) 这个代码，还是那个问题 nums=[3,3] ，target=6, return 的结果就是 [0.0]，而不是[0.1]，所以为了解决这个问题，就可以使用 pop 将这个 下标 对应的数值删掉，这样就会导致 nums的长度 会减少，下标p后面的索引 都会跟着 减 1，

​ if k>=p: k=k+1，是为了防止，p的数值删除，如果影响到了 k这个下标，即k在p的右边，就需要将 将k 进行+1，来返回实际上 nums中 k这个下标。

 上面我们也说过 如何 通过index 去取下标时，能避开已经 得到的下标，从而去取后续的下标，即nums[p]=123，就可以不需要代码  if k>=p:   k=k+1 了

回文数

将 x 的数据类型转化 为 字符串类型，然后使用双指针去依次判断，如果不相等就直接 返回 False 结束循环，如果想等就一直循环，直到条件 i<j 不满足，则返回 True,

首先要 排除一些特殊的情况 比如 x=负数 x= 10 x=0的情况，

然后开始计算 循环条件是 x > reverse_x，

remainder = x % 10 得到的是 x%10的余数

reverse_x = reverse_x * 10 + remainder 得到的是 x 从右往左的数据 ，（比如 x= 12345，那reverse_x 得到的就是 54，比如 x= 1234321，那reverse_x 得到的就是 1234）.

x = x // 10 则得到的是 每次 循环 x 都要 除于10 的结果，所以 x最终的值就是 x 除以 （10乘上循环的次数）

当条件不满足的时候

就需要判断 如果x是偶数 则只需要判断 x 是否等于 reverse_x（比如 x的原来值是 12344321，经过计算后 x为 1234 reverse_x 为 1234 ）

如果x是奇数，则需要先将reverse_x 除以10的结果跟 x 去判断 （比如 x的原来值是 1234321，经过计算后 x为 123 reverse_x 为 1234 ）

则返回结果
